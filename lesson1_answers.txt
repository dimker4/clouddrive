1. Как организовать клиент-серверное взаимодействие?
С использованием механизма сокетов
2. Как и в каком виде передавать файлы?
Можно передать сериализованный объект-оболочку для файла и десериализовать его на стороне сервера, либо передать массив байтов без сериализации
3. Как пересылать большие файлы?
Большие файлы быстрее передавать с помощью массива байтов, иначе объект-обертка получается тяжелым и занимает много ОЗУ
4. Как пересылать служебные команды?
а. При отправке с помощью сериализуемого класса, можно сделать еще один класс для служебных команд и на стороне сервера проверять к какому классу отновится полученый объект и в зависимости от результата, либо выполнять команду, либо обработать принятый файл.
б. При отправке массива байтов, необходимо выделить служебные байты в которых будет лежать информация о том, файл это или команда.
5.Что хранить в базе данных?
В БД можно хранить информацию для авторизации, можно хранить логи работы с файлами, для какого-нибудь анализа, при желании можно хранить всю структуру файлов и папок для каждого пользователя, не могу придумать пример когда это может пригодиться.
6. Как передавать структуру каталогов/файлов?
Возможно на стороне клиента структура должна архивироваться, а на стороне сервера разархивироваться в папку пользователя.
